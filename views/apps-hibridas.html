Aplicaciones híbridas

Una aplicación desarrollada como una página web que se compila dentro de un marco de aplicación nativa

WebView

Componente nativo que interpreta documentos HTML.
En otras palabras es un pequeño navegador.

Consideraciones

  • La aplicación se desarrolla como una página web, pero no es una página web.
  • Los recursos de HTML deben ser almacenada dentro de la aplicación. Debe evitarse en lo posible utilizar recursos directamente desde Internet.
  • El WebView no interpreta lenguaje del servidor (PHP, Ruby, Pyhton, Node.js, etc).
  • El dinamismo de la aplicación se trabaja desde JavaScript en el FrontEnd, nunca con leuguaje de servidor.

Petición HTTP

Funciones Nativas

  • Hardware: cámara, vibración, tonos, GPS, acelerómetro, etc
  • Servicios: SMS, MMS, Llamadas, etc
  • En el desarrollo de páginas web no tienen contempladas estas características
Desarrollar de interfaces de comunicación entre el documento HTML y el código nativo para cada plataforma.

Framework de desarrollo de aplicaciones móviles de código abierto que te permite usar tecnologías web estándares (HTML5, CCS3 y JavaScript) para desarrollos multiplataformas. Las aplicaciones se ejecutan dentro de contenedores específicos para cada plataforma y se basan API estándares compatibles para acceder a las características de cada dispositivo como sensores, datos, red, etc.
